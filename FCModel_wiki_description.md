== Обьектная модель формата Fidesys Case для Python ==

'''FCModel''' — это строго типизированная Python-библиотека для работы с файлами формата Fidesys Case (.fc). Библиотека предоставляет объектную модель для программного чтения, создания и модификации моделей конечных элементов, используемых в CAE-системе Fidesys.

=== Обзор ===

FCModel предназначена для автоматизации работы с моделями конечных элементов в формате Fidesys Case. Библиотека позволяет:
* Загружать существующие модели из файлов .fc
* Программно создавать новые модели
* Модифицировать параметры материалов, нагрузок, закреплений
* Сохранять изменения в новый файл формата .fc

Поддерживает Python 3.8 и выше.

=== Преимущества ===

FCModel предоставляет значительные преимущества по сравнению с прямым редактированием файлов формата [[Формат_Fidesys_Case|Fidesys Case]]:

* '''Строгая типизация''': Все сущности модели имеют четко определенные типы данных.
* '''Автоматическое кодирование бинарных данных''': Библиотека автоматически обрабатывает Base64-кодирование массивов координат, сеток и других бинарных полей.
* '''Объектная модель''': Вместо работы с вложенными JSON-структурами предоставляется удобная объектная модель с понятными методами и свойствами.
* '''Валидация данных''': Автоматическая проверка корректности данных при загрузке и сохранении файлов.
* '''Безопасность операций''': Типизированные операции предотвращают случайное повреждение структуры файла.

=== Архитектура ===

==== Основные классы ====

'''FCModel''' — корневой класс модели, содержащий все сущности:
* '''FCHeader''' — заголовок файла с метаданными
* '''FCMesh''' — сетка модели (узлы и элементы)
* '''FCBlock''' — блоки элементов с привязкой к материалам
* '''FCMaterial''' — материалы и их свойства
* '''FCLoad''' — нагрузки различного типа
* '''FCRestraint''' — закрепления и граничные условия
* '''FCCoordinateSystem''' — системы координат
* '''FCConstraint''' — контактные, связывающие и периодические ограничения
* '''FCReceiver''' — наборы приёмников результатов
* '''FCSet''' — узловые и граневые наборы

==== Поддерживаемые сущности ====

===== Типы элементов =====
Библиотека поддерживает все основные типы конечных элементов:
* Солидные элементы (FEM): TETRA4, TETRA10, HEX8, HEX20, WEDGE6, WEDGE15, PYR5, PYR13, QUAD4, QUAD8, TRI3, TRI6
* Солидные элементы (SEM): TETRA4S, TETRA10S, HEX8S, HEX20S, WEDGE6S, WEDGE15S, PYR5S, PYR13S, QUAD4S, QUAD8S, TRI3S, TRI6S
* Оболочечные элементы: MITC3, MITC6, MITC4, MITC8, SHELL3S, SHELL4S, SHELL6S, SHELL8S
* Балочные элементы: BEAM26, BEAM36, BEAM27, BEAM37, BEAM26S, BEAM36S, BEAM27S, BEAM37S
* Точечные массы: LUMPMASS3D, LUMPMASS6D, LUMPMASS2D, POINT3D, POINT2D, POINT6D

===== Материалы и свойства =====
* Группы свойств: elasticity (упругость), common (общие), thermal (тепловые), plasticity (пластичность), hardening (упрочнение), creep (ползучесть), preload (преднагружение), strength (прочность)
* Типы зависимостей: CONSTANT, TABULAR_X/Y/Z/TIME/TEMPERATURE/FREQUENCY/STRAIN/ELEMENT_ID/NODE_ID/MODE_ID, FORMULA

===== Нагрузки и граничные условия =====
* Типы нагрузок: давление, тепловой поток, конвекция, излучение, сосредоточенные силы, объёмные источники
* Типы закреплений: по перемещениям, скоростям, температурам, давлениям
* Начальные условия: перемещения, скорости, температуры, поровое давление

===== Ограничения =====
* Контактные ограничения: general, tied, tied_normal, tied_tangent
* Связывающие ограничения: elasticity, temperature, distance, pore_pressure, direction, interpolation
* Периодические ограничения: all, displacement, thermal_conduction, pore_pressure_conduction, velocity, acceleration

=== Использование ===

==== Установка ====

Библиотека выложена на pypl в открытый доступ. Вы можете установить её через pip:

<tt>
    pip install fc_model
</tt>

Или из исходников, скачав из репозитория [https://github.com/artmihant/fcmodel github].

==== Быстрый старт ====

<tt>
    from fc_model import FCModel
    # Загрузка модели из файла
    model = FCModel("path/to/model.fc")
    # ... модификация данных модели ...
    # Сохранение в файл
    model.save("path/to/output.fc")
</tt>

==== Пример модификации материала ====

<tt>
    from fc_model import FCModel, FCMaterial, FCMaterialProperty, FCData
    model = FCModel("case.fc")
    # Получение материала
    material_id = next(iter(model.materials))
    material = model.materials[material_id]
    # Добавление свойства плотности
    property_obj = FCMaterialProperty(
        type="USUAL",
        name="DENSITY",
        data=FCData(data="", dep_type=0, dep_data="")
    )
    material.properties.setdefault("common", [[]])[0].append(property_obj)
    model.save("case_updated.fc")
</tt>

=== Соответствие спецификации ===

Библиотека полностью соответствует спецификации формата Fidesys Case:
* Структура файла: JSON с бинарными вставками в Base64
* Кодировка: UTF-8
* Версия формата: 3
* Автоматическое кодирование/декодирование бинарных полей
* Поддержка всех разделов спецификации
